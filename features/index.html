Features

Everything Warlock ships

One framework, batteries included — from HTTP and real-time sockets to a MongoDB/Postgres ORM, S3 storage, a message bus, and a typed AI layer. Every capability links to its docs.

HTTP & API

The request layer.

6 capabilities

01 Routing & controllers Declarative routes, controllers, and use-cases. 02 Middleware Composable, ordered request middleware. 03 Validation Request validation backed by Seal schemas. 04 RESTful resources Resource controllers and typed API resources. 05 Rate limiting Throttle routes out of the box. 06 Modules & conventions Feature modules with batteries-included structure.

Data & ORM

One ORM, two databases.

4 capabilities

01 Unified ORM MongoDB · Postgres One typed model API that runs on either database. 02 Relations & migrations Model relations, migrations, and atomic operations. 03 Vector search Vectors Store and query embeddings for semantic search and RAG. 04 Transactions & aggregates Transactions, joins, and aggregation pipelines.

Real-time & Messaging

Events across your system.

4 capabilities

01 Real-time sockets Socket.IO Push real-time events to clients from the server. 02 Message bus RabbitMQ Publish/subscribe and request-respond over a broker. 03 Notifications Mail, in-app, and custom channels with preferences. 04 Mail SMTP · SES Transactional email over SMTP or Amazon SES.

Storage & Files

Files, anywhere.

4 capabilities

01 Unified storage S3-compatible One driver API over local disk or object storage. 02 File uploads Handle multipart uploads and stream to storage. 03 Image processing Sharp Resize, convert, and optimize images. 04 File-system utilities Atomic writes and directory management.

Caching

Fast by default.

4 capabilities

01 Multi-driver cache Redis · Postgres Memory, Redis, and Postgres drivers behind one API. 02 Tags, locks & SWR Tag invalidation, locks, and stale-while-revalidate. 03 Similarity search Semantic cache lookups for LLM responses. 04 Cache metrics Hit/miss metrics for observability.

AI

Agents on the same stack.

8 capabilities

01 Typed agents Agents with typed inputs and validated structured output. 02 Tools Define a tool with Seal — the schema is the contract. 03 Workflows & supervisors Compose agents into multi-step workflows with dynamic routing. 04 Orchestrators Sessions Stateful multi-turn sessions with durable state, resume, and compaction. 05 Planner Hand it a goal; the agent generates its own plan, then runs it. 06 Memory Working and semantic recall, wired into sessions. 07 Observability Panoptic Trace every run and export to OpenTelemetry or Langfuse. 08 Provider adapters 5 providers Swap between providers without touching agent code.

Auth & Access

Who, and what they can do.

2 capabilities

01 Authentication Tokens, route protection, and custom user types. 02 Authorization RBAC · ABAC Permission checks, policies, and roles.

Platform & DX

The operational backbone.

8 capabilities

01 Scheduler Cron jobs with retry, overlap prevention, and timezones. 02 Structured logging Channels, redaction, and shutdown flushing. 03 Typed schemas Seal Validation that doubles as types and JSON schema. 04 Request context Request-scoped context without prop-drilling. 05 Encryption Built-in encryption helpers. 06 CLI & generators Scaffold modules, commands, and connectors. 07 Testing Vitest Per-worker database and cache isolation. 08 Configuration Typed, layered application configuration.
